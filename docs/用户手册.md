## 1. 系统简介

本系统是一套**穿戴式 IMU 步态分析平台**，由 ESP32 WiFi 网关 + WT901WIFI 惯性传感器 + Python 桌面分析软件组成。可实现：

| 功能 | 说明 |
|------|------|
| **多传感器数据采集** | 同时采集多达 8 个 WT901WIFI 传感器的加速度/角速度/磁场数据 |
| **离线步态分析** | 基于 CSV 数据进行完整的步态参数提取（步速/步频/步长/步态周期/相位等） |
| **实时关节角度测量** | 双 IMU 相对姿态解算，实时显示膝关节/踝关节/髋关节的屈伸角度和曲线 |
| **结构化报告** | JSON 格式的完整步态分析报告 + PNG 姿态曲线 |

**适用场景**：临床步态评估、运动生物力学研究、康复训练监测。

## 2. 硬件准备

### 2.1 设备清单

| 设备 | 数量 | 说明 |
|------|------|------|
| ESP32 开发板 | 1 块 | 烧录 WiFi 网关固件 |
| WT901WIFI 传感器 | 2–8 个 | 穿戴式 9 轴 IMU（加速度+陀螺仪+磁力计） |
| USB 数据线 | 1 根 | 连接 ESP32 与电脑 |
| 绑带/魔术贴 | 若干 | 固定传感器于身体各部位 |

### 2.2 传感器佩戴位置

按照身体部位别名将传感器固定在对应位置：

| 别名 | 位置 | 所属关节 |
|------|------|----------|
| **S1** | 骶骨（后腰正中） | 骨盆参考 |
| **L4** | 左大腿前侧（膝盖上方约 10cm） | 左髋/左膝 |
| **L5** | 左小腿前侧（膝盖下方约 10cm） | 左膝/左踝 |
| **L6** | 左脚背 | 左踝 |
| **R4** | 右大腿前侧 | 右髋/右膝 |
| **R5** | 右小腿前侧 | 右膝/右踝 |
| **R6** | 右脚背 | 右踝 |

> **佩戴要点**：传感器 X 轴沿肢体长轴方向，Z 轴垂直于皮肤面向外。确保传感器与肢体紧密贴合，不松动。

### 2.3 连接方式

```
┌─────────────────┐     UDP (WiFi)      ┌──────────┐     USB Serial     ┌──────────┐
│ WT901WIFI 传感器 │ ──────────────────→ │ ESP32 AP │ ────────────────→ │  电脑    │
│ (多个, 佩戴于身体) │    192.168.10.x    │ WiFi网关  │   921600 baud     │ 运行软件  │
└─────────────────┘                     └──────────┘                   └──────────┘
```

1. 给 ESP32 上电，它会自动创建 WiFi 热点 `ESP32_Gait_Gateway`（密码 `12345678`）
2. 依次给每个 WT901WIFI 传感器上电，它们会自动连接到此热点
3. 用 USB 线将 ESP32 连接到电脑
4. ESP32 板上 LED（GPIO2）在有传感器连接时会亮起

## 5. 功能一：数据采集

数据采集用于**录制原始 IMU 数据到 CSV 文件**，供后续离线步态分析使用。

### 5.1 操作步骤

1. **选择串口**：在「连接」卡片中，从下拉框选择 ESP32 对应的 COM 口（通常带 CH340/CP2102 标识）。点击 `↻` 可刷新端口列表。

2. **确认波特率**：默认 `921600`，无需修改。

3. **点击 `▶ 开始采集`**：程序开始从串口读取 `[DATA]` 行，按设备 ID 自动分文件保存。

4. **观察状态**：标题栏会显示实时计数（如 `1,280 行 | 3 设备`），左侧状态行也会同步更新。

5. **点击 `■ 停止采集`**：停止录制。数据会自动保存到 `data/wt901_data_YYYYMMDD_HHMMSS/` 目录。

### 5.2 输出文件

```
data/wt901_data_20260625_143052/
├── WT5500010121.csv    ← 传感器 1 的全部数据
├── WT5500010131.csv    ← 传感器 2 的全部数据
└── ...                  ← 更多传感器
```

每个 CSV 文件包含 14 列：`device_id, timestamp, Acc_x, Acc_y, Acc_z, Gyr_x, Gyr_y, Gyr_z, Angle_x, Angle_y, Angle_z, Geo_x, Geo_y, Geo_z`

### 5.3 新建会话

点击 `⟳ 新建会话` 可清除当前结果和状态，准备新一轮采集。采集进行中此按钮不可用。

### 5.4 注意事项

- 数据采集与关节角度测量**互斥**，不能同时进行
- 开始采集前请确保传感器已全部上线（ESP32 LED 亮起）
- 如果串口断开，程序会自动尝试重连

## 6. 功能二：步态分析（离线）

步态分析对**已录制的 CSV 数据**进行完整的步态参数提取。

### 6.1 操作步骤

1. **选择数据目录**：
   - 方式 A：点击 `浏览...` 选择之前采集的数据文件夹
   - 方式 B：点击 `← 最近` 自动填入最近一次采集的目录
   - 方式 C：在「最近会话」列表中双击某一项

2. **检查传感器映射**：选择目录后，程序会自动扫描 CSV 文件并检测传感器。如果显示 `⚠ 未映射设备`，需要点击 `配置传感器映射...` 将设备 ID 映射到身体别名（详见[第 8 节](#8-传感器设备映射)）。

3. **选择工作模式**：

| 模式 | 所需传感器 | 说明 |
|------|-----------|------|
| `feet_only` | L6, R6 | 仅双脚分析，无转身检测 |
| `lower_body` | S1, L4, L5, L6, R4, R5, R6 | 下肢完整步态分析（推荐） |
| `upper_body` | H, T1, T12, L1-L3, R1-R3 | 上肢关节分析 |
| `full_body` | 全部 16 个 | 全身分析 |

4. **点击 `▶ 运行步态分析`**：分析开始，状态栏会显示进度条。通常耗时几秒到十几秒。

5. **查看结果**：分析完成后，右侧自动展示：
   - **上方 6 张汇总卡片**：步行速度、步频、步长、跨步时间、总步数、步态周期
   - **下方标签页**：基本参数、步态参数、相位参数、关节角度

### 6.2 输出文件

分析完成后 `output/` 目录生成：

| 文件 | 说明 |
|------|------|
| `attitude_<sensor>.png` | 每个传感器的俯仰/横滚/偏航姿态曲线 |
| `attitude_all_sensors_overview.png` | 所有传感器姿态叠图 |
| `gait_events_debug.png` | 步态事件检测调试图 |

### 6.3 导出结果

点击结果区域右上角的 `导出` 按钮，可将分析结果保存为 JSON 文件。

## 7. 功能三：实时关节角度测量

这是本系统的核心功能之一。通过在大腿和小腿上各佩戴一个 IMU 传感器，实时解算膝关节（或踝关节/髋关节）的**屈伸角度**，并在界面上以数值 + 实时曲线的方式呈现。

### 7.1 传感器佩戴（以膝关节为例）

```
            ┌──────────┐
 大腿 L4 →  │  WT901   │  (近端传感器 — 膝盖上方约 10cm)
            └──────────┘
                 │
     ┌───────────┤  ← 膝关节
     │           │
            ┌──────────┐
 小腿 L5 →  │  WT901   │  (远端传感器 — 膝盖下方约 10cm)
            └──────────┘
```

> **踝关节**: 小腿 L5/R5（近端） + 脚背 L6/R6（远端）
> **髋关节**: 骶骨 S1（近端） + 大腿 L4/R4（远端）

### 7.2 操作步骤

#### 第一步：展开关节卡片

点击左侧 `🦵 实时关节角度 (Joint Angle)` 标题展开卡片（如果已折叠）。

#### 第二步：选择关节

从「关节」下拉框中选择要测量的关节：
- `left_knee` — 左膝
- `right_knee` — 右膝
- `left_ankle` — 左踝
- `right_ankle` — 右踝
- `left_hip` — 左髋
- `right_hip` — 右髋

界面会自动更新近端/远端传感器的别名和身体部位标签。

#### 第三步：配置设备映射（首次使用）

每个 WT901 传感器有一个唯一的设备 ID（印在传感器外壳上，格式如 `WT901AB12345`）。

点击 `⚙ 配置设备映射`，将设备 ID 与身体别名（L4/L5 等）对应起来：

```
┌──────────────────────────────────────────────┐
│  将设备 ID (WT****) 映射到身体传感器别名       │
│                                              │
│  ┌──────────────────┬──────────┬────────┐   │
│  │ 设备 ID           │ 传感器别名 │ 操作   │   │
│  ├──────────────────┼──────────┼────────┤   │
│  │ WT901AB12345     │ [L4 ▼]   │ ✕ 删除  │   │
│  │ WT901AB67890     │ [L5 ▼]   │ ✕ 删除  │   │
│  └──────────────────┴──────────┴────────┘   │
│                                              │
│  [＋ 添加设备]   [填入 L4+L5]                 │
│  [保存映射]  [取消]                            │
└──────────────────────────────────────────────┘
```

> **提示**：点击「填入 L4+L5」可快速填充当前关节所需的两个别名。

映射只需配置一次，会持久化保存。之后程序会自动识别传感器。

#### 第四步：开始测量

1. 确保 ESP32 已通过 USB 连接，串口选择正确
2. 确保两个传感器已上电并连接到 ESP32 的 WiFi 热点
3. 点击 `▶ 开始测量`

程序会：
- 自动连接串口并等待传感器上线
- 检测到设备后，自动识别近端/远端传感器
- 收集 200 帧（约 4 秒）数据后，自动初始化 Mahony 姿态滤波器
- 初始化完成后提示「引擎已就绪，请保持静止站立后点击「开始校准」」

此时界面会自动切换到「关节角度」标签页，显示三条实时曲线面板。

#### 第五步：校准（必须执行！）

**每次测量都需要重新校准**，校准决定了角度的零点参考。

1. **保持静止站立姿势**（双腿直立，膝盖不弯曲）
2. 点击 `○ 开始校准`
3. 校准模式保持默认的「下肢站立校准」
4. 程序采集 3 秒数据，计算初始相对姿态 `q_rel_0`
5. 校准完成后，标定状态变为绿色的「已标定 ✓」

> **关键**：校准姿势决定了"零度"的含义。站立时膝盖伸直 = 0°。如果校准后坐下，膝盖弯曲 90°，界面就会显示约 90°。

#### 第六步：观察实时数据

校准完成后，界面会实时显示：

| 显示项 | 说明 |
|--------|------|
| **屈曲/伸展 (Flexion)** | 红色大字 — 膝关节主要运动角度 |
| **外展/内收 (Abduction)** | 绿色大字 — 侧向角度 |
| **内旋/外旋 (Rotation)** | 蓝色大字 — 轴向旋转角度 |
| **ROM** | 活动范围 = 最大值 − 最小值 |
| **实时曲线 ×3** | Flexion / Abduction / Rotation 三条曲线共享时间轴 |

每隔约 0.1 秒（10Hz 降采样显示）更新一次。

#### 第七步：停止测量

点击 `■ 停止测量`。数据会自动保存到 `joint_csv/joint_angle_YYYYMMDD_HHMMSS/` 目录：

```
joint_csv/joint_angle_20260625_150530/
├── joint_angle_data.csv    ← 全分辨率角度时序数据
└── session_info.json       ← 会话元数据
```

### 7.3 关节角度计算原理（简述）

```
近端 IMU (大腿) ──Mahony 滤波──→ q_prox (大腿姿态四元数)
远端 IMU (小腿) ──Mahony 滤波──→ q_dist (小腿姿态四元数)

q_rel_t     = inv(q_prox) × q_dist       (当前两传感器相对姿态)
q_joint_t   = inv(q_rel_0) × q_rel_t     (减去校准参考，得到关节姿态)
flexion_deg = euler_angles(q_joint_t)[roll]  → 屈伸角度
```

### 7.4 注意事项

- 校准时**必须保持完全静止**，否则校准会失败（程序会检测静止状态）
- 传感器需**牢固固定**，松动会导致角度震荡
- 测量过程中避免剧烈晃动传感器线缆
- 如果角度持续震荡超过 10°，请检查：传感器是否固定牢靠、WiFi 信号是否稳定、传感器电池是否充足

## 8. 传感器设备映射

### 8.1 两种映射

系统中有两种映射场景，注意区分：

| 特性 | 步态分析映射 | 关节设备映射 |
|------|------------|------------|
| **使用场景** | 离线 CSV 数据分析 | 实时串口关节测量 |
| **触发按钮** | 「配置传感器映射...」（步态分析卡片内） | `⚙ 配置设备映射`（关节角度卡片内） |
| **设备 ID 来源** | CSV 文件自动提取 | 手动输入 WT901 外壳编号 |
| **保存位置** | `data/wt901_data_*/device_alias_map.json` | `python/temp/device_alias_map.json` |

### 8.2 步态分析映射

当选择数据目录后，程序自动扫描 CSV 文件。如果存在无法识别的设备 ID（不在标准别名列表中），会在「传感器映射状态」处显示橙色警告。点击「配置传感器映射...」打开对话框，将设备 ID 映射到正确的身体别名。

### 8.3 关节设备映射

在关节测量中，传感器别名固定为 L4/L5 等。需要告诉程序"哪个设备 ID 对应 L4，哪个对应 L5"。

如果未配置映射，程序会按传感器**上线顺序自动分配**（第一个=近端，第二个=远端）。建议手动配置以确保正确。

## 10. 结果解读

### 10.1 汇总卡片（6 项核心指标）

| 指标 | 英文 | 含义 | 正常参考范围（成人） |
|------|------|------|---------------------|
| 步行速度 | Walking Speed | 平均行走速度 | 1.0–1.5 m/s |
| 步频 | Cadence | 每分钟步数 | 100–120 steps/min |
| 步长 | Step Length | 单步（一侧足跟着地到对侧足跟着地）的水平距离 | 0.5–0.8 m |
| 跨步时间 | Stride Time | 一个完整步态周期的时间 | 1.0–1.2 s |
| 总步数 | Total Steps | 采集期间有效步数总和 | — |
| 步态周期 | Gait Cycle | 平均步态周期时长 | 1.0–1.2 s |

### 10.2 基本参数标签页

包含步行速度、步频、跨步时间、总步数等，部分指标区分左右侧。

### 10.3 步态参数标签页

包含步长、跨步长、步时、抬脚高度等空间参数，区分左右侧。

### 10.4 相位参数标签页

包含支撑相（脚着地的时间占比，约 60%）、摆动相（脚在空中摆动的时间占比，约 40%）、双支撑相等时间参数。

### 10.5 关节角度标签页

- **离线分析模式**：显示所有已计算关节的角度范围和曲线
- **实时测量模式**：显示屈曲/伸展、外展/内收、内旋/外旋三条实时曲线的当前值

## 11. 常见问题

### Q1: 串口列表为空？

- 检查 ESP32 USB 线是否连接牢固
- 检查电脑是否安装了 CH340/CP2102 驱动
- 点击 `↻` 刷新端口列表

### Q2: 传感器没有数据？

- 确认传感器已上电（指示灯闪烁）
- 确认传感器已连接到 ESP32 WiFi 热点（ESP32 LED 亮起表示有客户端连接）
- 尝试重启传感器和 ESP32

### Q3: 关节角度一直为 0？

- 检查是否已完成自动初始化（状态显示「初始化:✓」）
- 检查是否已完成校准（状态显示「校准:✓」）
- 未标定时角度不更新曲线，需要在站立姿势下点击「开始校准」

### Q4: 关节角度大幅震荡（>10°）？

- 检查传感器是否牢固固定在肢体上
- 确保校准时完全静止站立
- 检查传感器电池电量
- 尝试重启测量流程

### Q5: 校准失败？

- 确保校准期间传感器完全静止（不要晃动身体）
- 如果持续失败，检查 `gyr_thr_dps` 阈值是否过于严格

### Q6: 步态分析运行失败？

- 确认数据目录中存在 CSV 文件
- 确认传感器映射完整（所有需要的别名都有对应文件）
- 确认选择的工作模式与可用传感器匹配

### Q7: 能否同时进行数据采集和关节测量？

不能。两者共用同一个串口，需要通过状态机互斥访问。如需同时记录，使用数据采集模式录制原始 CSV，然后用步态分析模式中的关节角度功能进行离线计算。
