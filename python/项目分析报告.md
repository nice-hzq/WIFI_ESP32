# 项目分析报告

## 1. 项目概述

本项目是一个**步态分析算法库**，使用 Python 实现，主要功能是处理穿戴式 IMU（惯性测量单元）传感器的运动数据，完成传感器校准、姿态估计、步态事件检测、时空参数计算、关节角度分析等任务，最终生成结构化的步态分析报告（JSON格式）。

项目设计目标是作为 C# 上位机程序的**算法后端**，通过 Python-C# 互操作方式被调用（`get_gait_analysis_object()` 和 `print_gait_summary()` 等函数供 C# 直接调用）。数据来源于 ROM_EVA 设备的多节点 IMU 传感器，每个节点通过蓝牙传输加速度、陀螺仪、磁力计/地磁数据，以 CSV 文件形式存储。

```bash

```

**方式二：运行步态分析管线主模块**
```bash
python -m gait.gait_pipeline
```

**方式三：被 C# 调用（生产环境）**
```csharp
// C# 通过 Python.NET 或进程调用
get_gait_analysis_object()   // 返回 GaitAnalysisResult 对象
print_gait_summary()         // 返回 JSON 字符串
```

### 4.3 运行前提

运行前需要配置以下 `config` 参数（在 `test_gait_report.py` 或调用方设置）：

- `config.tempDir` — 校准JSON文件目录（如 `./temp`）
- `config.originalDir` — 原始CSV数据目录（如 `./Data/walk_data1`）
- `config.curveDir` — 输出目录（如 `./output`）
- `config.WORK_MODE` — 工作模式：`"lower_body"` / `"upper_body"` / `"full_body"`
- `config.fs` — 采样率（默认50Hz）

## 5. 核心功能分析

### 5.1 已实现的功能模块

#### (1) 传感器数据读取与预处理
- 从目录批量读取 CSV 文件（按传感器别名自动识别）
- 支持三种工作模式：`upper_body`（9个上肢传感器）、`lower_body`（7个下肢传感器）、`full_body`（16个全身传感器）
- 传感器命名约定：H(头), T1/T12(躯干), L1-L6(左上肢/左下肢), R1-R6(右上肢/右下肢), S1(骶骨)

#### (2) 传感器校准
- **六面加速度校准**：采集 +X/-X/+Y/-Y/+Z/-Z 六面静止数据，计算加速度计 bias 和 scale
- **陀螺仪零偏校准**：基于静止数据计算陀螺仪零偏（deg/s）
- **磁力计椭球校准**：支持磁力计软铁/硬铁校准参数加载
- **T-pose 校准**：人体参考姿态校准
- 校准结果保存为 JSON 文件（`<alias>_gyro_bias.json`, `<alias>_acc_6face.json`, `<alias>_mag_calib.json`）
- 包含校准质量检查（阈值验证、异常检测）

#### (3) 姿态估计
- 使用 AHRS 库的 Mahony 互补滤波器进行姿态估计
- 四元数运算（乘法、归一化、共轭、逆、旋转、欧拉转换）
- 传感器坐标系到世界坐标系转换
- 重力去除
- IMU 安装姿态校正（INSTALL_POSE 参数）

#### (4) 步态事件检测
- 基于滤波后的脚部陀螺仪 X 轴数据检测 HS（Heel Strike，脚跟触地）
- 支持左/右脚独立检测
- 转身片段检测（基于腰部偏航角变化率）
- 转身期间计步

#### (5) 时空参数计算
- 时间参数：单步时间、跨步时间、步态周期、支撑相/摆动相/双支撑占比
- 空间参数：步长、步幅、步宽、行走距离、步速、步频
- 双检测器空间参数融合
- 足偏角（FPA）、足部滚动角、足部离地高度

#### (6) 关节角度分析
- **下肢**：髋关节屈曲/伸展、膝关节屈曲/伸展、踝关节背屈/跖屈（min/max）
- **上肢**：肩部 ROM（Range of Motion）、摆动速度
- 关节角度曲线输出（CSV格式）
- 关节角度+步态事件标注图输出（PNG格式）

#### (7) 报告生成
- 输出结构化 JSON 报告（`GaitAnalysisResult` 数据模型）
- 包含：基本参数、步态参数、相位参数、冲击参数、足部参数、关节极限数据
- 关节角度曲线图（含事件标记线）
- 步态周期均值/标准差 CSV 输出

## 8. 当前存在的问题

### 8.1 代码质量问题

1. **大量注释掉的旧代码**：`sensor/calibration.py` 中约 60% 的代码（600+行）为注释掉的旧版实现；`gait/gait_pipeline.py` 中约 300 行为注释掉的旧版 `create_gait_result()` 函数。这些严重降低代码可读性和可维护性。

2. **重复导入**：`sensor/calibration.py` 中多次重复 `import time`、`import numpy as np` 等语句（第24-32行），是明显的代码组织问题。

3. **空目录残留**：`curve/`、`lower_joint/`、`upper_joint/`、`Sensor_calib/`、`Config/` 均为空目录（仅含 `__pycache__`），无实际文件，造成目录结构混乱。

4. **双虚拟环境**：同时存在 `.venv`（Python 3.13）和 `venv`（Python 3.10），可能导致依赖版本不一致问题。

### 8.2 路径与配置问题

5. **混合路径分隔符**：代码中同时使用 `\\` 和 `/` 作为路径分隔符，在跨平台运行时可能出问题。`data_reader.py` 第16行有 `fix_path()` 函数做转换，但不统一。

6. **硬编码路径残留**：`config.py` 第19行有注释掉的硬编码绝对路径；`calibration.py` 第1178-1180行有注释掉的绝对路径。

7. **配置分散**：全局配置集中在 `core/config.py`，但参数过多（200+行），部分校准、滤波参数散落在各模块中。

### 8.3 功能完整性问题

8. **上肢分析部分完成**：上肢关节角度和躯干参数实际值多为0或None，仅在特定工作模式下可用。

9. **数据格式依赖强**：CSV文件命名和列名有严格约定，文件名必须包含传感器别名后缀（如 `_L6.csv`），如果命名不规范将无法识别。

10. **错误处理不完善**：部分函数在数据缺失时仅输出警告或返回None，调用链中缺少统一错误处理机制。

### 8.4 文档缺失

11. **无 README.md**：无项目说明、安装步骤、使用方法、数据格式要求等文档。
12. **无 API 文档**：供 C# 调用的接口函数无文档说明。
13. **代码注释极少**：除注释掉的代码外，活跃代码中几乎没有注释。

### 8.5 性能与维护
