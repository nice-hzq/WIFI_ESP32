# Claude Code 项目分析指令文档

> 文件用途：  
> 这是给 Claude Code 使用的第一阶段指令文档。  
> 你需要先阅读本文件，然后分析当前项目，最后生成一个新的 Markdown 项目分析报告。

---

## 一、你的任务

请你作为项目分析助手，先完整分析当前项目结构、代码文件、依赖关系和运行方式，然后在项目根目录下生成一个新的文件：

```text
项目分析报告.md
```

注意：

1. 不要只在聊天窗口输出分析结果。
2. 必须把最终分析结果写入 `项目分析报告.md` 文件。
3. 在完成项目分析报告之前，不要修改任何源代码。
4. 如果发现项目存在明显问题，可以在报告中指出，但不要直接改代码。
5. 如果项目较大，可以优先分析核心文件、入口文件、配置文件和主要业务目录。

---

## 二、分析范围

请重点分析以下内容：

### 1. 项目整体结构

请阅读当前项目的目录结构，并说明：

- 项目根目录下有哪些主要文件和文件夹
- 每个核心文件夹的作用
- 哪些文件是入口文件
- 哪些文件是配置文件
- 哪些文件是资源文件，例如图片、样式、数据等

---

### 2. 项目技术栈

请判断并总结本项目使用的技术，包括但不限于：

- HTML / CSS / JavaScript
- Python
- Vue / React / Node.js
- Flask / Django
- 其他框架或库
- 是否有前端、后端、数据库
- 是否需要安装依赖

如果无法确定，请在报告中说明“不确定”，不要随意猜测。

---

### 3. 项目运行方式

请根据项目文件判断如何运行该项目。

例如：

- 是否可以直接打开 `index.html`
- 是否需要运行本地服务器
- 是否需要执行 `npm install`
- 是否需要执行 `python xxx.py`
- 是否存在虚拟环境或依赖文件
- 是否存在 README、package.json、requirements.txt 等说明文件

如果当前项目缺少运行说明，请在报告中指出。

---

### 4. 核心功能分析

请根据代码和文件内容总结项目目前已经实现的功能。

需要说明：

- 当前项目主要做什么
- 已经实现了哪些页面或模块
- 每个页面或模块的主要作用
- 用户可以进行哪些操作
- 哪些功能可能还没有完成

---

### 5. 文件之间的关系

请分析主要文件之间的调用关系，例如：

- HTML 文件引用了哪些 CSS 文件
- HTML 文件引用了哪些 JavaScript 文件
- JavaScript 文件是否操作了页面元素
- 图片、数据或其他资源是否被正确引用
- 是否存在路径可能错误的问题

---

### 6. 当前项目存在的问题

请检查并总结当前项目可能存在的问题，例如：

- 文件结构混乱
- 命名不统一
- 图片路径错误
- 样式重复
- JavaScript 逻辑复杂或无效
- 代码中存在无用文件
- 页面移动端适配不足
- 运行方式不明确
- README 或说明文档缺失

注意：这里只需要分析问题，不要直接修改。

---

### 7. 后续开发建议

请根据项目现状，给出后续开发建议。

建议可以包括：

- 应该优先完善哪些页面
- 是否需要重构目录结构
- 是否需要新增 README.md
- 是否需要整理图片资源
- 是否需要优化移动端显示
- 是否需要添加部署说明
- 是否需要增加交互功能

建议要具体，不要写空话。

---

## 三、需要生成的新文件格式

请在项目根目录下生成：

```text
项目分析报告.md
```

该文件请按照以下结构编写：

```markdown
# 项目分析报告

## 1. 项目概述

## 2. 项目目录结构

## 3. 技术栈分析

## 4. 项目运行方式

## 5. 核心功能分析

## 6. 主要文件说明

## 7. 文件引用关系

## 8. 当前存在的问题

## 9. 后续开发建议

## 10. 下一步可执行任务清单
```

---

## 四、输出要求

生成 `项目分析报告.md` 时，请满足以下要求：

1. 使用中文。
2. 结构清晰，标题分级明确。
3. 不要写太多空泛内容。
4. 每个判断尽量基于实际项目文件。
5. 对不确定的内容要明确写“不确定”。
6. 不要编造不存在的文件或功能。
7. 不要直接修改项目代码。
8. 最后给出一个清晰的“下一步可执行任务清单”。

---

## 五、执行流程

请严格按照以下流程执行：

### 第一步：读取项目结构

先查看当前项目目录，包括根目录和主要子目录。

### 第二步：识别核心文件

重点查看：

- README.md
- package.json
- requirements.txt
- index.html
- main.js
- app.js
- style.css
- src 目录
- public 目录
- assets 目录
- images 目录
- data 目录

如果这些文件不存在，请根据实际项目结构继续分析。

### 第三步：分析项目内容

阅读核心代码文件，理解项目当前功能和运行方式。

### 第四步：生成项目分析报告

在项目根目录下创建并写入：

```text
项目分析报告.md
```

### 第五步：完成后告诉我

完成后，请在聊天窗口简短说明：

```text
已完成项目分析，并已生成 `项目分析报告.md`。
```

不要在聊天窗口重复输出完整报告内容。

---

## 六、特别注意

在生成 `项目分析报告.md` 之前，不要修改任何项目代码。

本阶段只做：

```text
项目分析 + 生成项目分析报告
```

不要做：

```text
代码重构
页面修改
新增功能
删除文件
移动文件
安装依赖
```

如果你认为项目需要修改，请只写入报告中的“后续开发建议”部分。
